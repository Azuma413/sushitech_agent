## Brief overview
- 本ガイドラインは、ユーザーとの協働開発における日本語でのコミュニケーションと、シンプルな実装を重視するプロジェクト向けのルールです。

## Communication style
- ユーザーへの応答やコード内コメントは日本語を使用すること
- 余計な会話や挨拶は避け、簡潔かつ技術的な説明を心がける
- 質問や追加情報の要求は必要最小限にとどめる

## Development workflow
- 実装はできるだけシンプルにし、パフォーマンスや品質は特別な指定がない限り重視しない
- 変更や作業内容は逐次ドキュメント化し、メモリバンクの更新を徹底する

## Coding best practices
- コードは冗長にならないよう簡潔に記述する
- コメントは日本語で要点のみ記載する
- ファイルや関数名はプロジェクトの既存命名規則に従う

## Project context
- メモリバンクの内容を最優先で参照し、プロジェクトの背景や進捗を常に把握する
- ルールやガイドラインは会話内容に基づき、一般的な推測や想像で追加しない

## Other guidelines
- .clinerulesディレクトリ内のファイル名はハイフン区切りとし、内容を端的に表す
- ルールファイルの内容はMarkdown形式で記述する
